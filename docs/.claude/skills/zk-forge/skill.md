# ZK-Forge 知识锻造者

一键完成机械工作，让用户专注于真正的思考。

## 核心理念

- **AI 做脚手架**：创建文件、提取定义、搜索双链、生成元数据
- **用户做思考**：填写"我的理解"区域（这是学习的核心）
- **效率与效果兼得**：节省 80% 时间，保留 100% 学习价值

## 执行流程

### 阶段 1：概念挖掘（来自 zk-miner）

1. 阅读用户指定的源文档
2. 识别 3-5 个最核心的原子概念
3. 确定笔记命名和类型

### 阶段 2：笔记创建

为每个概念创建独立的 Markdown 文件，使用以下模板：

```markdown
---
tags:
  - 主标签/子标签
aliases:
  - 别名1
  - 别名2
date: {{当前日期}}
type: permanent
status: seedling
source: "[[源文档名称]]"
---

# 概念名称

## 定义

> [从源文档提取的原始定义，用引用块标注]

## 核心要点

- 要点 1
- 要点 2
- 要点 3

## 💡 我的理解

<!--

## 🔗 关联笔记

- [[相关笔记1]] - 关系说明
- [[相关笔记2]] - 关系说明

## 📝 复习问题

> [!question] Q1
> 技术面试可能问到的问题

> [!question] Q2
> 深入理解需要思考的问题
```

### 阶段 3：双链编织（来自 zk-weaver）

1. 搜索 vault 中已有的相关笔记
2. 自动填充"关联笔记"区域
3. 标注关联类型（上层概念/下层实现/对比方案等）

### 阶段 4：元数据生成（来自 zk-architect）

1. 生成合适的标签层级
2. 创建别名（中英文、缩写）
3. 设计复习问题

## 输出格式

执行完成后，输出汇总报告：

```markdown
## 🔨 锻造完成

### 已创建笔记

| 笔记 | 类型 | 状态 |
|:-----|:-----|:-----|
| [[概念1]] | 核心定义 | ✅ 已创建 |
| [[概念2]] | 设计模式 | ✅ 已创建 |
| [[概念3]] | 配置策略 | ✅ 已创建 |

### 建立的链接

- [[概念1]] ↔ [[已有笔记A]]：上层概念
- [[概念2]] ↔ [[已有笔记B]]：对比方案

### ⏭️ 下一步

请打开以下笔记，填写"💡 我的理解"区域：
1. [[概念1]] - 预计 5 分钟
2. [[概念2]] - 预计 5 分钟
3. [[概念3]] - 预计 5 分钟

填写完成后，可使用 `/zk-tutor` 获取反馈。
```

## 用户交互

### 触发方式

```
/zk-forge
```

默认处理 `<current_note>` 中的文档。

### 可选参数

- 指定输出目录：`/zk-forge --output Zettelkasten/AI`
- 指定概念数量：`/zk-forge --count 3`
- 仅分析不创建：`/zk-forge --dry-run`

## 与其他技能的关系

| 技能 | 关系 | 何时单独使用 |
|:-----|:-----|:-------------|
| `/zk-miner` | 已整合 | 只想识别概念，不创建文件 |
| `/zk-weaver` | 已整合 | 为已有笔记补充双链 |
| `/zk-architect` | 已整合 | 为已有笔记补充元数据 |
| `/zk-tutor` | 保持独立 | 填写"我的理解"后获取反馈 |

## 注意事项

- **不替代思考**："我的理解"区域必须由用户填写
- **保持原子性**：每个笔记只记录一个概念
- **尊重已有结构**：不覆盖 vault 中已存在的同名笔记
- **相对路径**：所有文件操作使用 vault 内相对路径
