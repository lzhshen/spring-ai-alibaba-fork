# Tool Calling (函数调用)

Tool Calling 允许 AI 模型根据当前对话上下文，自动决定何时以及如何调用外部工具或 API。

## 工作流程
1. **定义工具**: 开发者将 Java 方法声明为 `@Bean` 并使用 `@Description` 补充描述。
2. **注册工具**: 在 `ChatClient` 调用时声明需要开启的工具。
3. **模型转换**: LLM 判断需要调用工具时，返回特定 JSON。
4. **自动执行**: Spring AI 自动反射执行对应 Java 方法。
5. **结果返回**: 将方法执行结果喂回给 LLM 以生成最终回答。

## 扩展实现
- **Python Tool**: 支持调用部署在外部环境的 Python 脚本。
- **API 工具库**: 内置了一系列常用业务工具的封装。

## 注意事项
- **描述的准确性**: `@Description` 应尽量清晰，直接影响 LLM 的判断。
- **异常处理**: 工具执行过程中的异常应被妥善捕获，否则会直接中断 AI 推理流程。
